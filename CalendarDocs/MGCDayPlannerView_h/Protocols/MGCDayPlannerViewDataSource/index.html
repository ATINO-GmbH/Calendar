MGCDayPlannerViewDataSource

Extends Protocol:
NSObject
Declared In:
MGCDayPlannerView.h

Introduction

An object that adopts the MGCDayPlannerViewDataSource protocol is responsible for providing the data and views required by a day planner view.


Methods

-dayPlannerView:canCreateNewEventOfType:atDate:

Asks the data source if an event can be created with given type and date.

-dayPlannerView:canMoveEventOfType:atIndex:date:toType:date:

Asks the data source if the specified event can be moved to given date, or change its type. If the method returns NO, a forbidden sign is shown to indicate that the view cannot be moved to that date and, if dropped, the delegate method dayPlannerView:moveEventOfType:atIndex:date:toType:date: won't be called.

-dayPlannerView:createNewEventOfType:atDate:

Informs the data source that a new event was dragged and dropped at given date. Data source should update the event accordingly, and ask the day planner view to reload events.

-dayPlannerView:dateRangeForEventOfType:atIndex:date:

Asks the data source for the date range of the specified event in the day planner view. (required)

-dayPlannerView:moveEventOfType:atIndex:date:toType:date:

Informs the data source that an event was dragged and dropped to a different date / time. Data source should update the event accordingly, and ask the day planner view to reload events.

-dayPlannerView:numberOfEventsOfType:atDate:

Asks the data source for the number of events of given type at specified date. (required)

-dayPlannerView:shouldStartMovingEventOfType:atIndex:date:

Asks the data source if the specified event can be moved around. If the method returns YES, the event view can be dragged and dropped to a different date / time.

-dayPlannerView:viewForEventOfType:atIndex:date:

Asks the data source for the view that corresponds to the specified event in the collection view. (required)

-dayPlannerView:viewForNewEventOfType:atDate:

Asks the data source for the view to be displayed when a new event is about to be created.

dayPlannerView:canCreateNewEventOfType:atDate:

@optional

- (BOOL)dayPlannerView:(MGCDayPlannerView*)view 
        canCreateNewEventOfType:(MGCEventType)type atDate:(NSDate*)date;

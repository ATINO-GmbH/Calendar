Introduction

You can use an instance of MGCDayPlannerView to display events as a schedule.

This view displays a grid with days as columns and time slots as rows. User can scroll infinitely through days or swipe through pages of several days. Events are displayed in cells, which are subclasses of `MGCEventView` and can contain any subviews according to the type of the event.

The view also displays a bar at the top with full-day events.

dateAtPoint:rounded:

Returns the date at the specified point in the day planner view. 

- (NSDate*)dateAtPoint:(CGPoint)point rounded:(BOOL)rounded;
Parameters
point

A point in the day planner view’s coordinate system.

rounded

If set to YES, the returned date is rounded to the nearest 15-minutes slot.

Return Value

The date at the specified point, or nil if the date can't be determined.

Discussion

If `point` lies in the timed-event part, then the returned date contains information about day and time. If `point` lies outside the timed-event part, like within the header or all-day events part, then the returned date contains only day/month/year information (time part is set to 00:00).

dequeueReusableViewWithIdentifier:forEventOfType:atIndex:date:

Returns a reusable event view cell object located by its identifier. 

- (MGCEventView*)dequeueReusableViewWithIdentifier:(NSString*)identifier 
        forEventOfType:(MGCEventType)type atIndex:(NSUInteger)index 
        date:(NSDate*)date;
Parameters
identifier

A string identifying the cell object to be reused. This parameter must not be nil.

type

The type of event for which the view is requested.

index

The index of the event.

date

The date of the event.

Return Value

A valid MGCEventView object.

Discussion

Call this method from your data source object when asked to provide a new event view for the day planner. This method dequeues an existing view if one is available or creates a new one based on the class you previously registered.

WARNING:

You must register a class using the registerClass:forEventViewWithReuseIdentifier: method before calling this method.

scrollToDate:options:animated:

Scrolls the view until a certain date is visible. 

- (void)scrollToDate:(NSDate*)date options:(MGCDayPlannerScrollType)options 
        animated:(BOOL)animated;
Parameters
date

The date to scroll into view. It will be the first visible date on the left of the view.

options

Specify if scrolling through dates only (MGCDayPlannerScrollDate), time only (MGCDayPlannerScrollTime), or both (MGCDayPlannerScrollDateTime).

animated

Specify YES to animate the scrolling behavior or NO to adjust the visible content immediately.

WARNING:

If `date` param is not in the scrollable range of dates, an exception is thrown.

WARNING:

If the view is already curently scrolling, this will have no effect.

pagingEnabled

Determines whether paging is enabled for this day planner view. 

@property(nonatomic,
    getter=isPagingEnabled) BOOL pagingEnabled;
Discussion

If the value of this property is YES and the number of visible days is equal or greater than 7 (i.e. the view is showing at least a week), the day planner view stops on the current calendar starting day of the week when the user scrolls. If set to YES but the view is showing less than a week, it stops on multiples of the view’s bounds. If paging is enabled, the view also snaps to day boundaries. The default value is YES.
